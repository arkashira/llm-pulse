 # Customer-Journey.md

## Aware

### Trigger Event
- User experiences a significant drop in the performance of their LLM API, leading to increased latency or errors in their applications.
- User notices a change in the output of their applications that rely on the LLM API, such as a decrease in accuracy or an increase in incorrect predictions.

### Friction Points
- Lack of awareness about the existence of an independent alert service for LLM API performance degradation and model drift.
- Difficulty in identifying the root cause of the performance issue, leading to wasted time and resources.

### User Emotions
- Frustration and annoyance due to the impact on their applications and the time spent trying to resolve the issue.
- Concern about the potential loss of revenue or customers due to the performance degradation.

### Opportunities to Delight
- Providing clear and actionable alerts that help the user quickly identify the root cause of the performance issue.
- Offering detailed performance analytics and insights that help the user understand the impact of the performance degradation on their applications.

### Metric
- Awareness level among potential users (e.g., number of sign-ups, website visits, social media engagement).

## Consider

### Trigger Event
- User becomes aware of llm-pulse as a potential solution to their performance issue.
- User conducts research and compares llm-pulse with other similar services.

### Friction Points
- Lack of understanding of how llm-pulse works and how it can help resolve their performance issue.
- Concerns about the cost and complexity of implementing llm-pulse.

### User Emotions
- Skepticism about the effectiveness of llm-pulse in resolving their performance issue.
- Anxiety about the potential cost and complexity of implementing llm-pulse.

### Opportunities to Delight
- Providing clear and concise documentation that explains how llm-pulse works and how it can help resolve their performance issue.
- Offering a free trial or demo to allow the user to test llm-pulse before committing to a purchase.

### Metric
- Number of sign-ups for the free trial or demo.
- Number of support tickets or inquiries related to understanding how llm-pulse works.

## Try

### Trigger Event
- User signs up for the free trial or demo of llm-pulse.
- User sets up llm-pulse to monitor their LLM API.

### Friction Points
- Difficulty in setting up llm-pulse, such as configuring the API keys or integrating it with their existing infrastructure.
- Lack of clear instructions or guidance on how to use llm-pulse effectively.

### User Emotions
- Excitement about the potential benefits of using llm-pulse to resolve their performance issue.
- Frustration if they encounter any issues during the setup process.

### Opportunities to Delight
- Providing clear and concise setup instructions and guidance.
- Offering responsive and helpful customer support to assist with any issues that arise during the setup process.

### Metric
- Number of successful setups.
- Number of support tickets or inquiries related to the setup process.
